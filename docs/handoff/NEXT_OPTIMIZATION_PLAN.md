# 后续功能和交互优化计划

更新时间：2026-05-27 23:08

## 当前两条路线

现在项目里保留两套仪表盘：

```text
自定义卡片版：/minecraft-home-card-test/home
html-pro-card 版：/minecraft-html-pro
```

两条路线不是互相替代，而是适合不同目标：

```text
自定义卡片版：适合长期产品化、组件复用、可视化编辑器。
html-pro-card 版：适合快速生成、现场配置、复制分享。
```

## 第一优先级

### 自定义卡片版

```text
首页常用设备改为可配置列表
设备页增加筛选模式：楼层、房间、类型、状态
房间页升级为楼层 > 房间结构
场景执行增加确认和执行反馈
自动化页显示启用/禁用状态
```

### html-pro 生成器

```text
一键同步后直接跳转打开新仪表盘
同步前检测 path 是否已存在并提示覆盖
自动推荐结果显示推荐分数和原因
支持保存/导入多个映射方案
支持从当前 HA dashboard 反向读取配置
```

## 第二优先级

### 功能扩展

```text
支持窗帘 cover
支持风扇 fan
支持扫地机 vacuum
支持摄像头 camera 的快照入口
支持净水器、水质、滤芯等传感器
支持空气质量、新风、地暖、热水器
支持安防状态总览
```

### 数据聚合

```text
按楼层统计在线设备
按房间统计开启设备
按设备类型统计状态
按楼层统计功率
按房间统计能耗
展示离线实体和异常实体
```

## 第三优先级

### 视觉和交互

```text
移动端导航优化
首页组件密度调优
加载态、空态、离线态统一
状态色系统统一
卡片点击反馈更快
危险动作二次确认
长列表折叠和展开
动画降级选项，适合低性能平板
```

### 工程维护

```text
拆分 minecraft-device-card.js 的 CSS 和渲染函数
为分组识别逻辑加独立测试
为 html-pro 生成器抽出共享实体映射配置
生成器支持 dry-run 和远端同步日志
把部署脚本标准化，减少手工 SFTP 操作
```

## 建议下一步

下一轮建议先做一个小闭环：

```text
html-pro 生成器：同步成功后自动打开新仪表盘 + 覆盖确认
自定义卡片版：设备页筛选模式
```

这两个改动能最明显提升日常使用体验：一个让生成更顺，一个让卡片版看大量实体时更可控。
